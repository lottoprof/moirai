# common: security

Базовый чек-лист безопасности. Полные правила — в
`rules/security.md`.

## Секреты

- Никаких секретов в коде, конфигах, документации, тестах.
- Локально — `.dev.vars` (gitignored).
- Production — `wrangler pages secret put <NAME>`.
- Никогда не логировать значения секретов.

## Web Crypto

- Только `crypto.subtle` для шифрования / подписей / хэшей.
- AES-GCM с уникальным IV на каждое значение.
- HMAC / sign-verify через `crypto.subtle.sign` / `verify`.

## Валидация

- Любой внешний ввод (params, query, body, headers, cookies, события
  webhook) — через схему (zod / эквивалент) до использования.
- Cookie / JWT — проверять подпись и срок до доверия содержимому.

## Защищённая зона

- Все `src/pages/app/**` и защищённые `src/pages/api/**` — за
  guard'ом (middleware или явный check в начале handler'а).
- Сессии — opaque-токены в KV, не угадываемые.
- CSRF / SameSite — настроить явно.

## Зависимости

- Новые пакеты — edge-compat-чек (`skills/js-ts/deps.md`).
- Не тащить пакеты с известными CVE без обоснования.
- Lock-файл коммитится, чтобы dependency не подменилась.

## Логирование и ошибки

- Не возвращать stack trace / внутреннюю структуру в ответ
  пользователю.
- Не логировать заголовки авторизации, тела запросов с
  чувствительными данными.
- Сообщения об ошибке — короткие, без утечки.

## Если сомневаешься — выбирай безопаснее.

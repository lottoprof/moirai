# Architecture Rules (moirai)

## Слои и зоны

Проект разделён на **публичный слой** и **защищённую зону (ЛК)**.
Это разделение — архитектурный инвариант, и оно отражается в путях,
правилах гидрации и в распределении кода между агентами.

```
src/
├── pages/
│   ├── *.astro                  # публичные SEO-страницы
│   ├── app/**                   # защищённая зона (auth-gated)
│   └── api/**                   # серверные эндпоинты (SSR/API)
├── components/
│   ├── public/**                # vanilla, CSS-only
│   └── app/**                   # islands, Vidstack, runtime JS
├── layouts/
│   ├── public/**
│   └── app/**
├── lib/
│   ├── server/**                # edge-only код (биндинги, crypto.subtle)
│   └── shared/**                # изоморфные утилиты
├── middleware.ts                # auth/ratelimit на границе app
├── styles/**
└── env.d.ts                     # типы биндингов
```

## Правила слоёв

1. **Публичный слой — без JS-фреймворков и островов.** Только
   vanilla `<script>` (если совсем нужно) и CSS-only анимации.
   Никаких `client:*` директив. Это требование SEO/perf.
2. **Защищённая зона — острова разрешены.** Используем
   `client:idle` / `client:visible` по умолчанию. `client:load`
   — только если функциональность нужна сразу при отрисовке.
   Vidstack живёт здесь.
3. **Серверный код — только в `src/lib/server/` и `src/pages/api/`.**
   Любой код, использующий `Astro.locals.runtime.env` (D1/KV/R2),
   принадлежит серверному слою.
4. **Изоморфный код — в `src/lib/shared/`.** Edge-compatible, без
   Node API даже косвенно (см. `edge-compat.md`).

## Рендеринг и адаптер

- Adapter: `@astrojs/cloudflare`.
- Режим SSR (`output: "server"` или `"hybrid"`) фиксируется
  отдельным решением в `decisions.md` при первом скаффолде.
- Биндинги читаются исключительно через
  `Astro.locals.runtime.env.<NAME>`. Глобальные `process.env` не
  использовать — они недоступны в Workers runtime.
- Типы биндингов генерируются `wrangler types` и кладутся
  в `worker-configuration.d.ts` (или путь, заданный в
  `wrangler.toml`). Не редактировать вручную.

## Хранилища

- **D1** — реляционные данные (если выбрано). Все изменения
  схемы — через `schema/migrations/NNNN_*.sql`. См.
  `.agent/agents/schema.md`.
- **KV** — сессии, токены, кэш короткоживущих значений.
- **R2** — крупные бинарные артефакты (медиа, статические
  загрузки) — S3-совместимый API.

Биндинги определяются в `wrangler.toml`. Имена — `SCREAMING_SNAKE_CASE`.

## Источник истины

- Архитектурные решения — `.agent/rules/decisions.md` (manifest)
  и `decisions_archive.md` (полные записи).
- Концепция и сценарии — `docs/` (создаётся по мере нужды).
- README — точка входа для нового разработчика.

Изменения в архитектурно значимых документах должны сохранять
согласованность между ними. Новые предположения помечать явно как
предположения, а не как гарантированное поведение.

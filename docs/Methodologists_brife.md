# Бриф для методистов

## Что мы строим

Платформу для онлайн-курса по кинорежиссуре. Студент покупает программу,
проходит её по фиксированному расписанию, на каждый практический модуль
получает 1:1 сессию с преподавателем.

## Что нужно от вас

Описать программы **Beginner** и **Intermediate** в виде структурированной
таблицы. Каждый модуль — отдельная строка с атрибутами.

Файл для заполнения — `Moirai_Modules_Template.xlsx` (приложен к этому
брифу). Внутри три листа:

- **Инструкция** — подробные правила и описание колонок
- **Модули** — основная рабочая таблица (заполнять здесь)
- **Справочник** — допустимые значения для выпадающих списков
- **Сводка** — автоматическая статистика по программам (обновляется по мере
  заполнения)

В шаблоне выпадающие списки и проверка значений — Excel сам подскажет если
что-то введено неправильно.

## Главные принципы

**Атомарная единица — модуль.** Программа — это последовательность модулей.
Один модуль = одна тема, один связный блок учебного материала.

**Длительность всей программы получится автоматически.** Не задавайте её
отдельно. Система сложит дни всех модулей и сама посчитает общую длительность.

**Длительность модуля задаётся в днях.** Это сколько времени студенту
даётся на работу с материалом до следующей сессии. Разные модули — разная
длительность: где-то 3 дня хватит, где-то нужно 7-10.

**Модули бывают теоретические и практические.**

- **Практические** (с домашкой) — после такого модуля студент делает работу
  и встречается с преподавателем на 1:1 сессии (50 минут). Колонка
  «Домашка» = «Да».
- **Теоретические** (без домашки) — короче, без сессии. Колонка «Домашка»
  = «Нет». Несколько теоретических модулей могут идти подряд.

## Структура сессии (для понимания)

50 минут, 1:1 со студентом, три части:

1. **10 минут** — разбор предыдущей домашки. Письменные правки преподаватель
   уже отправил студенту до сессии — на сессии обсуждаются устно.
2. **20-25 минут** — углубление по новому модулю. **Записывается**, попадает
   в личный кабинет студента как дополнение к тексту/видео модуля.
3. **10-15 минут** — практическое упражнение по только что пройденному.
   Не записывается.

Это не повторение текста модуля, а его **углубление и применение** к работе
конкретного студента.

## Что заполнять в каждой строке

Подробное описание каждой колонки — на листе «Инструкция» в шаблоне. Здесь
короткая версия:

- **ID** — короткий уникальный код, формат `программа-номер-название`,
  например `beg-01-visual-language`
- **Программа** — Beginner или Intermediate
- **Трек** — Directing / Editing / Scriptwriting / Producing
- **Порядок** — целое число, начиная с 1
- **Название** (на русском, перевод сделают переводчики)
- **Краткое описание** — 1-2 предложения
- **Тип содержания** — Текст / Видео / Текст+видео
- **Длительность** — в днях
- **Домашка** — Да или Нет
- **Описание домашки** — если «Да»
- **Цели модуля** — что студент будет уметь после
- **Ключевые понятия** — термины через запятую

В шаблоне уже есть **3 примера** заполненных строк с пометкой «можно
удалить». Они показывают как должна выглядеть готовая запись.

## Что **не нужно** делать сейчас

- **Не пишите тело модулей.** Сейчас нам нужна только структура программы:
  какие модули, в каком порядке, с какими атрибутами. Сами тексты модулей,
  упражнения, prompt'ы домашек — это отдельный этап после.
- **Не указывайте даты.** Конкретные даты появятся только когда будет
  объявлен запуск программы. Длительность задаётся в днях, не в датах.
- **Не описывайте Bundle.** Bundle = Beginner + Intermediate. Отдельно его
  заполнять не нужно.

## Когда возникают сомнения

«Это один модуль или два?» — если темы слабо связаны или для них нужна
разная домашка, лучше два.

«Сколько дней дать на модуль?» — отталкивайтесь от того, что реально нужно
сделать студенту. Прочитать текст и сделать упражнение — 3-5 дней. Снять
сцену — 5-7 дней. Смонтировать короткий фильм — 10-14 дней.

«Нужна ли домашка?» — да, если это практика. Нет, если теория, история,
обзор.

## Что после

Заполнили → отправили нам. Мы:

1. Проверяем консистентность (порядок, ID без дубликатов, валидные значения)
2. Импортируем в систему
3. Возвращаемся к вам за телом каждого модуля поочерёдно

После импорта в систему длительность программы будет автоматически
посчитана (например: 12 модулей × разные дни = 78 дней программы), и можно
объявлять первый запуск.

## Размер программ — для ориентира

Текущий маркетинговый план обещает:

- **Beginner**: программа на ~12 недель с 12 практическими сессиями.
  Это значит примерно **12 модулей с домашкой** + теоретические модули
  между ними.
- **Intermediate**: ~14 недель с 14 практическими сессиями.

Это ориентир, не жёсткое требование. Если педагогически правильнее 11 или
13 — берите как правильно. Маркетинговое описание поправим под реальную
программу, не наоборот.

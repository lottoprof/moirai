# /works и /journal — стратегические разделы

> Инструкция: роль публичных разделов `/{locale}/works` (галерея
> студенческих работ) и `/{locale}/journal` (блог) в воронке Moirai,
> их обоснование, anatomy страниц, связь с основным продуктом.

См. также:
- `Architecture.md` §6 (структура сайта), §10 (resources), §11 (agent
  workflow для journal)
- `Home_page_SEO.md` (SEO-стратегия главной)
- `voice-guide.md` (бренд-голос, обязателен для journal-агентов)

---

## 1. `/works` — Галерея студенческих работ

### 1.1 Что это

Публичная коллекция короткометражных фильмов, сделанных выпускниками
программ. **Социальное доказательство** — то, что студенты реально
снимают, не маркетинговое обещание.

### 1.2 Зачем

**Социальное доказательство до покупки.**
Посетитель видит: вот люди прошли программу и сделали кино. Это
снимает основное возражение "получится ли у меня".

**SEO-актив.**
Каждая работа = отдельная страница `/{locale}/works/[slug]` с
описанием, режиссёром, годом, программой. Schema.org `VideoObject` —
попадание в Google video search.

**Удержание выпускников.**
Их фильм лежит на сайте Moirai с credits — мотивация делать
качественно, делиться с друзьями.

**Внешние ссылки.**
Выпускники сами линкуют свои работы из портфолио → обратные ссылки
на наш домен.

### 1.3 Что внутри страницы каждой работы

- Видео (R2 публичный bucket `media.moirai.film/works/`)
- Название, режиссёр, год
- Короткий synopsis (1–2 абзаца)
- Из какой программы выпустился студент → ссылка на программу
- Опционально: cast, длительность, фестивальные показы

### 1.4 Где живёт

- **Метаданные + описание:** Content Collection
  `/src/content/works/[slug].{locale}.mdx`
- **Видео:** R2 публичный bucket
- **Доступ:** публично без auth

## 2. `/journal` — Блог

### 2.1 Что это

Регулярно публикуемые статьи о кино, режиссуре, монтаже, сценарии.
**Главный SEO-driver на 12–18 месяцев.**

### 2.2 Зачем

**Long-tail SEO.**
Главная страница и страницы программ берут крупные коммерческие
запросы (`online filmmaking program`), но их объём ограничен и
конкуренция высокая. Long-tail запросы (`how to direct a dialogue
scene`, `three-act structure explained`, `editing rhythm
fundamentals`) — миллионы в сумме, низкая конкуренция, высокий
buying intent: человек ищет ответ → находит наш разбор → видит, кто
это написал → читает про программу → подаёт заявку.

**Authority building.**
Регулярные качественные разборы делают Moirai авторитетом в теме.
Через 12 месяцев имена lead-инструкторов должны быть знакомыми для
тех, кто гуглит filmmaking education.

**Internal linking.**
Из каждого поста ссылки на программы, инструкторов, конкретные
модули — распределение link equity внутри сайта.

**Material для соцсетей.**
Каждый пост = 5–10 постов в Instagram/YouTube короткими нарезками.

### 2.3 Архитектурный контекст

Журнал — **единственная зона agent-driven workflow** в проекте (см.
`Architecture.md` §10, §11).

**Pipeline:**

```
бриф (тема, keywords)
    ↓
draft
    ↓
translate (en + ru пара)
    ↓
review человеком
    ↓
publish
```

Агенты (Claude Code, Codex) пишут драфты по `/voice-guide.md` — но
никто не видит, что это делал агент. **Authorship** — реальные имена
lead-инструкторов / Moirai team как редакторов.

### 2.4 Почему именно agent-driven

Журнал **не работает**, если посты выходят раз в месяц (что реально
для инструкторов при их основной работе). Agent-driven позволяет
публиковать 1 пост в неделю на обоих языках качественно.

### 2.5 Где живёт

- **Content Collection** `/src/content/journal/[slug].{locale}.mdx`
- **Структура поста (frontmatter):** title, slug, description,
  published_at, author, category, tags, cover_image, locale,
  translation_id

## 3. Воронка: связь между этими страницами и продуктом

```
Случайный человек гуглит "how to write a dialogue scene"
              ↓
        Попадает на /journal/dialogue-scenes
              ↓
        Читает разбор + видит "by <lead instructor name>"
              ↓
        Из поста — ссылки на /instructors/<id> + /beginner
              ↓
        Кликает программу → видит /works (другие студенты сняли это)
              ↓
        Apply
```

### 3.1 Роль каждой страницы в воронке

| Страница | Работает на | Roll |
|---|---|---|
| **Home** | Кто уже знает, что им нужна программа | Top-of-funnel, immediate conversion |
| **/journal** | Кто только ищет информацию (в 100× больше) | Attraction, authority |
| **/works** | Кто колеблется ("я-то смогу?") | Социальное доказательство перед Apply |

### 3.2 Почему все три необходимы

Каждая из трёх — **необходимое звено** воронки.

- `/works` без `/journal` — социальное доказательство без attraction
  (некому показывать)
- `/journal` без `/works` — трафик без conversion (приходят, но не
  верят, что сами смогут)
- Без обоих — воронка работает только на тех, кто уже решил подать
  заявку (тонкая верхушка)
